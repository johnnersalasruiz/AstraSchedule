Fundamentos

¿Qué es un Agente IA?

Un LLM normal responde preguntas. Un agente IA recibe un objetivo, decide cómo lograrlo, usa herramientas, observa los resultados y repite hasta terminar — de forma autónoma.

LLM simple vs Agente IA — diferencia fundamental
👷
Analogía perfecta: un LLM es como un libro muy inteligente — sabe muchísimo pero no puede hacer nada solo. Un agente IA es como un empleado nuevo muy inteligente — no solo sabe cosas, sino que recibe una tarea, busca la información que necesita, usa las herramientas disponibles (teléfono, computador, base de datos) y entrega un resultado concreto.
🎯
Orientado a objetivos
Recibe un objetivo en lenguaje natural ("analiza estas notas y dime qué temas me faltan") y trabaja hasta lograrlo.
🔧
Usa herramientas
Puede llamar APIs, leer archivos, consultar bases de datos, buscar en internet — cualquier Tool MCP disponible.
🧠
Razonamiento multi-paso
Divide problemas complejos en pasos. Si un paso falla, lo detecta y busca una alternativa.
💾
Memoria de sesión
Recuerda lo que hizo en pasos anteriores dentro de una conversación para tomar mejores decisiones.
Tipos

Tipos de Agentes IA

Existen diferentes patrones de diseño para agentes. En esta guía usaremos principalmente ReAct — el más implementado y el más fácil de entender.

Ciclo ReAct (Reason + Act) — el patrón más usado
Tipo Cómo funciona Mejor para Complejidad
ReActRazona → actúa → observa, en cicloTareas conversacionales con tools⬤ Baja
Plan & ExecutePrimero crea un plan completo, luego ejecuta cada pasoTareas largas y predecibles⬤ Media
Multi-agenteVarios agentes especializados se coordinanProyectos grandes con subtareas⬤ Alta
Con MemoriaPersiste contexto entre sesiones usando BD o archivosAsistentes personales, tutores⬤ Media
✅
En esta guía usaremos ReAct. Es el patrón más común, el más fácil de entender y el que usan la mayoría de agentes en producción (incluido cómo funciona internamente Claude cuando usa tools MCP).

Arquitectura

Cómo se conecta un Agente con MCP

El agente es el cerebro — usa el LLM para razonar y decide qué tools MCP llamar. Las tools son los brazos — ejecutan acciones en el mundo real.

Arquitectura completa — Agente IA + Tools MCP
🔑
Punto clave: el agente NO llama directamente a las APIs externas. Siempre pasa por el servidor MCP que ya construiste en las guías anteriores. Esto hace el sistema modular — puedes cambiar la fuente de datos sin modificar el agente.

// ═══════════════════════════════════════════════════════════════
// agent-news.js — Agente Periodista de Noticias
// ───────────────────────────────────────────────────────────────
// Descripción : Agente ReAct que busca noticias RSS en tiempo real
//               y genera un artículo periodístico automáticamente.
// Modelo      : Llama 3.1 70B vía Groq (100% gratuito)
// Tools MCP   : buscar_noticias(), obtener_noticias_recientes()
// Fuentes RSS : BBC Mundo · TechCrunch · El País Tech
// Patrón      : ReAct — razona → actúa → observa → responde
// ───────────────────────────────────────────────────────────────
// Requisitos previos:
//   npm install dotenv rss-parser
//   .env con: GROQ_API_KEY=gsk_...
// ───────────────────────────────────────────────────────────────
// Uso: node agent-news.js
//      Cambia el objetivo en la última línea del archivo
// ═══════════════════════════════════════════════════════════════

import 'dotenv/config';
import Parser from 'rss-parser';

// ── Configuración del modelo Groq (100% gratis)
const GROQ_KEY   = process.env.GROQ_API_KEY;
const GROQ_MODEL = 'llama-3.1-70b-versatile';
const parser     = new Parser({ timeout: 8000 });

// ── Tools MCP disponibles para el agente
// (simulamos el servidor RSS directamente aquí para simplicidad)
const FEEDS = {
  bbc:        'https://feeds.bbci.co.uk/mundo/rss.xml',
  techcrunch: 'https://techcrunch.com/feed/',
  elpais:     'https://feeds.elpais.com/mrss-s/pages/ep/site/elpais.com/section/tecnologia/portada'
};

// ── Definición de tools que el agente puede llamar
const TOOLS = [
  {
    type: 'function',
    function: {
      name: 'buscar_noticias',
      description: 'Busca noticias por keyword en feeds RSS de BBC Mundo, TechCrunch y El País Tech',
      parameters: {
        type: 'object',
        properties: {
          keyword: { type: 'string', description: 'Término a buscar en las noticias' },
          cantidad: { type: 'number', description: 'Número de noticias a retornar (máximo 5)' }
        },
        required: ['keyword']
      }
    }
  },
  {
    type: 'function',
    function: {
      name: 'obtener_noticias_recientes',
      description: 'Obtiene los últimos N titulares de una fuente específica',
      parameters: {
        type: 'object',
        properties: {
          fuente: { type: 'string', enum: ['bbc','techcrunch','elpais','todas'] },
          cantidad: { type: 'number' }
        },
        required: ['fuente']
      }
    }
  }
];

// ── Implementación real de las tools
async function ejecutarTool(nombre, args) {
  // buscar_noticias: filtra por keyword en todos los feeds
  if (nombre === 'buscar_noticias') {
    const kl = (args.keyword || '').toLowerCase();
    const resultados = [];
    for (const [k, url] of Object.entries(FEEDS)) {
      const feed = await parser.parseURL(url);
      feed.items
        .filter(i => (i.title+' '+(i.contentSnippet||'')).toLowerCase().includes(kl))
        .slice(0, args.cantidad || 3)
        .forEach(i => resultados.push({
          fuente: k, titulo: i.title,
          descripcion: (i.contentSnippet||'').slice(0,150),
          fecha: i.pubDate, url: i.link
        }));
    }
    return JSON.stringify({ total: resultados.length, noticias: resultados });
  }

  // obtener_noticias_recientes: trae los N más recientes de una fuente
  if (nombre === 'obtener_noticias_recientes') {
    const keys = args.fuente === 'todas' ? Object.keys(FEEDS) : [args.fuente];
    const items = [];
    for (const k of keys) {
      const feed = await parser.parseURL(FEEDS[k]);
      feed.items.slice(0, args.cantidad || 3)
        .forEach(i => items.push({ fuente: k, titulo: i.title, url: i.link }));
    }
    return JSON.stringify(items);
  }

  return JSON.stringify({ error: 'Tool desconocida' });
}

// ── Motor del agente ReAct
async function llamarGroq(mensajes) {
  const res = await fetch('https://api.groq.com/openai/v1/chat/completions', {
    method: 'POST',
    headers: { 'Authorization': `Bearer ${GROQ_KEY}`, 'Content-Type': 'application/json' },
    body: JSON.stringify({
      model: GROQ_MODEL,
      messages: mensajes,
      tools: TOOLS,
      tool_choice: 'auto',
      max_tokens: 2000
    })
  });
  return (await res.json()).choices[0].message;
}

export async function correrAgente(objetivo) {
  console.log(`\n🤖 Agente iniciado: "${objetivo}"\n`);

  const mensajes = [
    {
      role: 'system',
      content: `Eres un agente periodista inteligente. Tu trabajo es:
1. Usar las tools disponibles para buscar noticias relevantes
2. Analizar los resultados obtenidos
3. Redactar un resumen periodístico claro y en español
Siempre usa las tools primero antes de generar el artículo final.`
    },
    { role: 'user', content: objetivo }
  ];

  // Ciclo ReAct: razona → actúa → observa → repite
  let pasos = 0;
  while (pasos < 5) {
    pasos++;
    console.log(`  ⚙️  Paso ${pasos}...`);

    const respuesta = await llamarGroq(mensajes);
    mensajes.push(respuesta);

    // ¿El LLM quiere llamar una tool?
    if (respuesta.tool_calls?.length) {
      for (const tc of respuesta.tool_calls) {
        const args = JSON.parse(tc.function.arguments);
        console.log(`  🔧 Llamando: ${tc.function.name}(${JSON.stringify(args)})`);
        const resultado = await ejecutarTool(tc.function.name, args);
        mensajes.push({ role: 'tool', tool_call_id: tc.id, content: resultado });
      }
      continue; // vuelve a llamar al LLM con los resultados
    }

    // ¿El LLM generó la respuesta final?
    if (respuesta.content) {
      console.log(`\n✅ Resultado final:\n`);
      console.log(respuesta.content);
      return respuesta.content;
    }
  }
}

// ── Punto de entrada — cambia el tema aquí
correrAgente("Busca y analiza las 3 noticias más importantes sobre inteligencia artificial de esta semana");

JavaScript · Gemini 1.5 Flash + Notas Markdown 
// ═══════════════════════════════════════════════════════════════
// agent-tutor.js — Agente Tutor de Apuntes Académicos
// ───────────────────────────────────────────────────────────────
// Descripción : Agente ReAct que lee notas Markdown y actúa como
//               tutor personal: responde preguntas, detecta temas
//               faltantes y crea nuevas notas automáticamente.
// Modelo      : Gemini 1.5 Flash vía Google AI Studio (100% gratis)
// Tools       : listar_notas(), leer_nota(), crear_nota(),
//               buscar_en_notas()
// Formato API : Google Generative Language (diferente a OpenAI)
// ───────────────────────────────────────────────────────────────
// Requisitos previos:
//   npm install dotenv
//   .env con: GEMINI_API_KEY=AIzaSy...
//   carpeta notas/ con archivos .md
// ───────────────────────────────────────────────────────────────
// Uso: node agent-tutor.js
//      Cambia la pregunta en la última línea del archivo
// ═══════════════════════════════════════════════════════════════

import 'dotenv/config';
import { readFileSync, writeFileSync, readdirSync, existsSync, mkdirSync } from 'fs';
import { join } from 'path';

const GEMINI_KEY = process.env.GEMINI_API_KEY;
const GEMINI_URL = `https://generativelanguage.googleapis.com/v1beta/models/gemini-1.5-flash:generateContent?key=${GEMINI_KEY}`;
const DIR_NOTAS  = './notas';
if (!existsSync(DIR_NOTAS)) mkdirSync(DIR_NOTAS);

// ── Tools disponibles
const TOOLS_DEF = {
  functionDeclarations: [
    {
      name: 'listar_notas',
      description: 'Lista todas las notas disponibles con sus nombres y tamaño en caracteres',
      parameters: { type: 'OBJECT', properties: {} }
    },
    {
      name: 'leer_nota',
      description: 'Lee el contenido completo de una nota por su nombre',
      parameters: {
        type: 'OBJECT',
        properties: { nombre: { type: 'STRING', description: 'Nombre del archivo .md' } },
        required: ['nombre']
      }
    },
    {
      name: 'crear_nota',
      description: 'Crea una nueva nota Markdown con el contenido indicado',
      parameters: {
        type: 'OBJECT',
        properties: {
          nombre:    { type: 'STRING', description: 'Nombre del archivo sin .md' },
          contenido: { type: 'STRING', description: 'Contenido en formato Markdown' }
        },
        required: ['nombre', 'contenido']
      }
    },
    {
      name: 'buscar_en_notas',
      description: 'Busca un término en todas las notas y retorna los párrafos donde aparece',
      parameters: {
        type: 'OBJECT',
        properties: { termino: { type: 'STRING' } },
        required: ['termino']
      }
    }
  ]
};

// ── Implementación de tools
function ejecutarTool(nombre, args) {
  if (nombre === 'listar_notas') {
    const archivos = readdirSync(DIR_NOTAS).filter(f => f.endsWith('.md'));
    return JSON.stringify(archivos.map(f => ({
      archivo: f,
      chars: readFileSync(join(DIR_NOTAS,f),'utf-8').length
    })));
  }
  if (nombre === 'leer_nota') {
    const path = join(DIR_NOTAS, args.nombre);
    if (!existsSync(path)) return JSON.stringify({ error: 'Nota no encontrada' });
    return readFileSync(path, 'utf-8');
  }
  if (nombre === 'crear_nota') {
    const file = args.nombre.endsWith('.md') ? args.nombre : args.nombre+'.md';
    writeFileSync(join(DIR_NOTAS,file), args.contenido, 'utf-8');
    return JSON.stringify({ ok: true, archivo: file });
  }
  if (nombre === 'buscar_en_notas') {
    const kl = args.termino.toLowerCase();
    const res = [];
    readdirSync(DIR_NOTAS).filter(f=>f.endsWith('.md')).forEach(f=>{
      const txt = readFileSync(join(DIR_NOTAS,f),'utf-8');
      if(txt.toLowerCase().includes(kl)) {
        const lineas = txt.split('\n').filter(l=>l.toLowerCase().includes(kl));
        res.push({ archivo:f, lineas });
      }
    });
    return JSON.stringify(res);
  }
}

// ── Llamada a Gemini API
async function llamarGemini(partes) {
  const res = await fetch(GEMINI_URL, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      contents: partes,
      tools: [TOOLS_DEF],
      systemInstruction: {
        parts: [{ text: `Eres un tutor académico que ayuda estudiantes de Ingeniería de Sistemas.
Tienes acceso a las notas del estudiante. Cuando respondan preguntas:
1. Primero lista las notas disponibles si no las conoces
2. Lee las notas relevantes para dar una respuesta precisa
3. Si el tema no está en las notas, créa una nota nueva con la información
4. Siempre cita qué nota usaste en tu respuesta
Responde siempre en español.` }]
      }
    })
  });
  return (await res.json()).candidates[0].content;
}

export async function correrTutor(pregunta) {
  console.log(`\n📚 Tutor iniciado: "${pregunta}"\n`);

  const historial = [{ role:'user', parts:[{text:pregunta}] }];
  let pasos = 0;

  while (pasos++ < 6) {
    console.log(`  ⚙️  Paso ${pasos}...`);
    const respuesta = await llamarGemini(historial);
    historial.push({ role:'model', parts:respuesta.parts });

    // ¿Gemini quiere llamar una tool?
    const toolCalls = respuesta.parts.filter(p => p.functionCall);
    if (toolCalls.length) {
      const resultadosTools = [];
      for (const tc of toolCalls) {
        const { name, args } = tc.functionCall;
        console.log(`  🔧 Llamando: ${name}(${JSON.stringify(args)})`);
        const result = ejecutarTool(name, args);
        resultadosTools.push({ functionResponse: { name, response: { result } } });
      }
      historial.push({ role:'user', parts:resultadosTools });
      continue;
    }

    // ¿Respuesta final de texto?
    const textoFinal = respuesta.parts.find(p => p.text)?.text;
    if (textoFinal) {
      console.log(`\n✅ Respuesta:\n${textoFinal}`);
      return textoFinal;
    }
    break;
  }
}

// ── Prueba el agente
correrTutor("¿Qué diferencia hay entre useEffect y useState en React? Revisa mis notas y dame un resumen");

agent-sql.js
JavaScript · Groq + SQLite · Text-to-SQL 
// ═══════════════════════════════════════════════════════════════
// agent-sql.js — Agente Analista de Datos (Text-to-SQL)
// ───────────────────────────────────────────────────────────────
// Descripción : Agente ReAct que recibe preguntas en español,
//               genera queries SQL automáticamente, las ejecuta
//               en SQLite y explica los resultados en lenguaje
//               natural con recomendaciones.
// Modelo      : Llama 3.1 70B vía Groq (100% gratuito)
// Tools       : ejecutar_sql(), listar_tablas()
// BD          : uniajc.db (crear primero con: node seed-db.js)
// ───────────────────────────────────────────────────────────────
// Requisitos previos:
//   npm install dotenv better-sqlite3
//   .env con: GROQ_API_KEY=gsk_...
//   node seed-db.js  (ejecutar ANTES que este archivo)
// ───────────────────────────────────────────────────────────────
// Uso: node agent-sql.js
//      Cambia la pregunta en la última línea del archivo
// ═══════════════════════════════════════════════════════════════

import 'dotenv/config';
import Database from 'better-sqlite3';

const GROQ_KEY   = process.env.GROQ_API_KEY;
const GROQ_MODEL = 'llama-3.1-70b-versatile';
const db         = new Database('uniajc.db');

// ── Schema para que el LLM sepa qué tablas y columnas existen
const SCHEMA = `
Tablas disponibles en la BD uniajc.db:

estudiantes(id, nombre, apellido, codigo, programa, semestre, ciudad, promedio, email)
materias(id, codigo, nombre, creditos, semestre, programa)
notas(id, estudiante_id, materia_id, nota, periodo)
docentes(id, nombre, email, programa, tipo)  -- tipo: 'planta' o 'catedra'

Valores de programa: 'Ingeniería de Sistemas', 'Administración'
`;

const TOOLS = [
  {
    type: 'function',
    function: {
      name: 'ejecutar_sql',
      description: 'Ejecuta una query SQL SELECT en la base de datos UNIAJC y retorna los resultados',
      parameters: {
        type: 'object',
        properties: {
          query: { type: 'string', description: 'Query SQL SELECT válida' },
          descripcion: { type: 'string', description: 'Qué hace esta query en una frase' }
        },
        required: ['query']
      }
    }
  },
  {
    type: 'function',
    function: {
      name: 'listar_tablas',
      description: 'Muestra todas las tablas y columnas disponibles en la base de datos',
      parameters: { type: 'object', properties: {} }
    }
  }
];

async function ejecutarTool(nombre, args) {
  if (nombre === 'ejecutar_sql') {
    try {
      const rows = db.prepare(args.query).all();
      console.log(`  📊 SQL: ${args.descripcion || args.query}`);
      console.log(`     → ${rows.length} filas`);
      return JSON.stringify({ filas: rows, total: rows.length, query: args.query });
    } catch(e) {
      return JSON.stringify({ error: e.message });
    }
  }
  if (nombre === 'listar_tablas') { return SCHEMA; }
}

async function llamarGroq(mensajes) {
  const res = await fetch('https://api.groq.com/openai/v1/chat/completions', {
    method: 'POST',
    headers: { 'Authorization': `Bearer ${GROQ_KEY}`, 'Content-Type': 'application/json' },
    body: JSON.stringify({
      model: GROQ_MODEL, messages: mensajes,
      tools: TOOLS, tool_choice: 'auto', max_tokens: 2000
    })
  });
  return (await res.json()).choices[0].message;
}

export async function correrAnalistaSql(pregunta) {
  console.log(`\n🗄️  Analista SQL iniciado: "${pregunta}"\n`);
  const mensajes = [
    {
      role: 'system',
      content: `Eres un analista de datos de la UNIAJC (Institución Universitaria Antonio José Camacho de Cali, Colombia).
Tienes acceso a la base de datos académica. El schema es:\n${SCHEMA}
Cuando el usuario haga una pregunta:
1. Genera el SQL correcto para responderla
2. Ejecuta la query con la tool ejecutar_sql
3. Analiza los resultados y explica en lenguaje natural
4. Si aplica, da recomendaciones o hallazgos interesantes
Siempre muestra el SQL que usaste para transparencia.
Responde en español.`
    },
    { role: 'user', content: pregunta }
  ];

  let pasos = 0;
  while (pasos++ < 6) {
    const resp = await llamarGroq(mensajes);
    mensajes.push(resp);
    if (resp.tool_calls?.length) {
      for (const tc of resp.tool_calls) {
        const r = await ejecutarTool(tc.function.name, JSON.parse(tc.function.arguments));
        mensajes.push({ role:'tool', tool_call_id:tc.id, content:r });
      }
      continue;
    }
    if (resp.content) {
      console.log(`\n✅ Análisis:\n${resp.content}`);
      return resp.content;
    }
    break;
  }
}

correrAnalistaSql("¿Cuántos estudiantes hay por programa y cuál es el promedio de notas en cada uno?");

🎓 Proyecto Integrador de Curso
Agente de Optimización de Horarios — Facultad de Ingeniería UNIAJC
La Facultad de Ingeniería de UNIAJC enfrenta cada semestre el reto de distribuir horarios académicos de forma manual — un proceso tedioso, propenso a conflictos y difícil de optimizar. Este proyecto propone desarrollar un agente conversacional basado en IA que automatice y optimice ese proceso, detecte conflictos y notifique a los actores involucrados.
🤖 Agente ReAct 🗄️ SQLite 🆓 Groq o Gemini gratis 📋 Entrega por sprints 👥 Equipos de 3-4 estudiantes
📥
Insumos del Agente
Carga académica de docentes, matrículas y prerrequisitos de estudiantes, disponibilidad de aulas físicas y virtuales por franja horaria.
⚙️
Procesamiento
Detección de conflictos (choque de aulas, docente en dos lugares, prerrequisito no cumplido), generación de propuestas válidas optimizando disponibilidad.
📤
Resultados
Horario validado en JSON/tabla, reporte de conflictos detectados, notificación a directores de programa, exportación a CSV.
🎯
Impacto Esperado
Menos tiempo operativo para directores, mayor equidad en carga docente, mejor experiencia académica y proceso reproducible cada semestre.

//   grupos              — grupos a programar con inscritos
//   horario_asignado    — resultado del agente (empieza vacío)
//   conflictos_detectados — log de conflictos del agente

// ⚠️ El agente (agent-horarios.js) lo construye tu equipo.

import Database from 'better-sqlite3';
const db = new Database('horarios.db');

db.exec(`
  -- ═══════════════════════════════════════════════════
  -- TABLA: docentes
  -- Quién puede dictar clases y en qué franjas
  -- ═══════════════════════════════════════════════════
  CREATE TABLE IF NOT EXISTS docentes (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    nombre      TEXT NOT NULL,
    email       TEXT UNIQUE,
    tipo        TEXT CHECK(tipo IN ('planta','catedra')) NOT NULL,
    carga_max   INTEGER DEFAULT 20,   -- horas/semana máximo
    carga_actual INTEGER DEFAULT 0    -- se actualiza al asignar
  );

  -- ═══════════════════════════════════════════════════
  -- TABLA: disponibilidad_docente
  -- Qué días y franjas puede un docente
  -- ═══════════════════════════════════════════════════
  CREATE TABLE IF NOT EXISTS disponibilidad_docente (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    docente_id  INTEGER REFERENCES docentes(id),
    dia         TEXT CHECK(dia IN ('Lunes','Martes','Miercoles','Jueves','Viernes','Sabado')),
    franja      TEXT CHECK(franja IN ('6-8','8-10','10-12','14-16','16-18','18-20','20-22'))
  );

  -- ═══════════════════════════════════════════════════
  -- TABLA: materias
  -- ═══════════════════════════════════════════════════
  CREATE TABLE IF NOT EXISTS materias (
    id           INTEGER PRIMARY KEY AUTOINCREMENT,
    codigo       TEXT UNIQUE NOT NULL,
    nombre       TEXT NOT NULL,
    creditos     INTEGER NOT NULL,
    horas_semana INTEGER NOT NULL,   -- generalmente = creditos
    programa     TEXT NOT NULL,
    semestre     INTEGER NOT NULL,
    prerequisito_id INTEGER REFERENCES materias(id)  -- puede ser NULL
  );

  -- ═══════════════════════════════════════════════════
  -- TABLA: aulas
  -- Aulas físicas y virtuales disponibles
  -- ═══════════════════════════════════════════════════
  CREATE TABLE IF NOT EXISTS aulas (
    id         INTEGER PRIMARY KEY AUTOINCREMENT,
    codigo     TEXT UNIQUE NOT NULL,
    tipo       TEXT CHECK(tipo IN ('presencial','virtual','laboratorio')),
    capacidad  INTEGER NOT NULL,
    bloque     TEXT       -- ej: 'Bloque A', 'Bloque B', 'Virtual'
  );

  -- ═══════════════════════════════════════════════════
  -- TABLA: grupos
  -- Cada grupo de estudiantes que debe recibir una materia
  -- ═══════════════════════════════════════════════════
  CREATE TABLE IF NOT EXISTS grupos (
    id           INTEGER PRIMARY KEY AUTOINCREMENT,
    materia_id   INTEGER REFERENCES materias(id),
    num_grupo    INTEGER NOT NULL,   -- ej: 1, 2, 3
    cupo_max     INTEGER NOT NULL,
    inscritos    INTEGER DEFAULT 0,
    programa     TEXT NOT NULL,
    semestre     INTEGER NOT NULL
  );

  -- ═══════════════════════════════════════════════════
  -- TABLA: horario_asignado
  -- El resultado final — un registro por cada clase
  -- ═══════════════════════════════════════════════════
  CREATE TABLE IF NOT EXISTS horario_asignado (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    grupo_id    INTEGER REFERENCES grupos(id),
    docente_id  INTEGER REFERENCES docentes(id),
    aula_id     INTEGER REFERENCES aulas(id),
    dia         TEXT NOT NULL,
    franja      TEXT NOT NULL,
    periodo     TEXT NOT NULL,        -- ej: '2025-1'
    estado      TEXT DEFAULT 'propuesto'  -- propuesto | confirmado | conflicto
  );

  -- ═══════════════════════════════════════════════════
  -- TABLA: conflictos_detectados
  -- Registro de cada conflicto encontrado por el agente
  -- ═══════════════════════════════════════════════════
  CREATE TABLE IF NOT EXISTS conflictos_detectados (
    id           INTEGER PRIMARY KEY AUTOINCREMENT,
    tipo         TEXT NOT NULL,  -- 'aula_ocupada' | 'docente_cruce' | 'prerequisito' | 'carga_excedida'
    descripcion  TEXT NOT NULL,
    entidad_id   INTEGER,         -- el ID del elemento con conflicto
    resuelto     INTEGER DEFAULT 0,
    fecha        TEXT DEFAULT (date('now'))
  );
`);

Lo que el equipo debe implementar: la función del agente (agent-horarios.js) con el LLM y las tools. El seed de datos ya está dado arriba. Las tools mínimas requeridas son:

detectar_conflictos(periodo) — consulta la tabla horario_asignado buscando cruces de aula o docente en el mismo día/franja.
verificar_disponibilidad(docente_id, dia, franja) — consulta disponibilidad_docente.
asignar_clase(grupo_id, docente_id, aula_id, dia, franja) — inserta en horario_asignado si no hay conflicto.
listar_grupos_sin_horario() — retorna los grupos que aún no tienen asignación.
